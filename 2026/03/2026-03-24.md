## Top Stories

### 1. How to Do AI-Assisted Engineering — 15 Engineers Share Their Real Workflows
*Gregor Ojstersek, Engineering Leadership*

Fifteen engineers and engineering leaders — from startups to large companies — share how they're actually using AI in their day-to-day engineering work. The dominant pattern: spend more time on design (up to 50% of project time) because AI can handle implementation so fast that architecture becomes the bottleneck. Multiple engineers run Claude Code agents in parallel on different codebase sections. A key practice is asking AI to review its own output ("Review this like a senior engineer"), which consistently catches issues the generation pass missed. One engineer found nine concrete issues in a five-minute AI review pass, saving hours of rework. The consensus on tools: Claude Code (Opus 4.6) as the primary interactive partner, OpenAI Codex for background automated analysis, and Claude running as a CI reviewer on every PR. The counterintuitive lesson: AI doesn't reduce the number of iterations — it increases them, because each iteration is faster and you can afford more. The skill shift is from writing code to reviewing it.

### 2. When Your Vibe Coded App Goes Viral — And Then Goes Down
*Every (Dan Shipper)*

Dan Shipper built Proof, an agent-native document editor, mostly as a side project in about 10 days. It launched, went viral, and immediately crashed under load — requiring 24 hours of sleepless debugging with Codex sub-agents. Key findings from the postmortem: 140,000 lines of code, 1,600 commits, 600+ PRs in ~10 days of part-time work (the median full-time dev commits 417 times/year). What still breaks with vibe coding: AI agents prefer local patches over root-cause fixes, models don't know best practices for newer libraries, and the fix→deploy→monitor feedback loop is slow on complex codebases. The key bottleneck isn't code generation — it's diagnosis speed. "An experienced engineer narrows the hypothesis space based on years of failed experiments. AI agents can only run the experiment and wait." The allocation economy: the scarce resource is knowing *where* to direct intelligence, not the intelligence itself.

### 3. How Agentic RAG Works
*ByteByteGo*

Standard RAG fails in three scenarios: ambiguous queries it can't rewrite, answers scattered across multiple documents it can't combine, and false confidence when retrieved content is relevant-looking but wrong. Agentic RAG fixes this by turning the one-shot pipeline into a control loop. The agent can evaluate whether retrieval was good enough, route queries to different knowledge sources, rewrite queries that came back with weak results, and synthesize from multiple retrievals before generating. The architecture mirrors the ReAct pattern: reason → act → observe → repeat. Trade-offs are real: a standard RAG query takes 1-2 seconds; an agentic one with three loops can take 10+ seconds and cost 3-10x more. The evaluator paradox is the biggest risk — you're trusting one LLM call to judge whether another LLM's retrieval was sufficient. Use Agentic RAG for complex multi-source queries; don't use it for simple FAQ lookups where the knowledge base is clean.

### 4. OpenAI Acquires Astral (uv/ruff/ty) — What It Means for Python and the AI Wars
*Simon Willison*

OpenAI acquired Astral, the company behind uv (Python environment manager), ruff (linter/formatter), and ty (type checker), folding the team into Codex. uv had 126 million downloads last month — it's now load-bearing Python infrastructure. Simon Willison's analysis is nuanced: the acquisition is simultaneously about talent (Astral has elite Rust engineers including BurntSushi of ripgrep fame) and about strengthening Codex. The competitive framing is stark: Anthropic acquired Bun (JavaScript runtime, now a key Claude Code component) in December 2025, and both companies are fighting intensely for the $200/month developer subscription market. The risk: OpenAI could use uv ownership as competitive leverage. The reassurance: Astral's tools are permissively licensed and "forkable" in a worst-case scenario. Also in this issue: GPT-5.4 nano at $0.20/M input tokens can describe your entire 76,000-photo library for $52.

### 5. Why Vibe Coding to Production Fails (And Qwen's Team Implodes)
*Amazing CTO (Stephan Schmidt)*

The reason engineers can vibe-code to production but non-engineers can't: "Semantic Anchors." Terms like "abstraction layer," "TDD," and "rate limiting" are trigger words that make AI do the right thing — and developers know them, non-developers don't. Meanwhile, the Qwen team is hemorrhaging resignations following a re-org that put a Google Gemini hire in charge, exactly when Qwen 3.5 models are peaking. This raises the trillion-dollar question: if open-source models (Qwen3.5-9B hits 93.8% of GPT-5.4 on a local MacBook M5) match frontier performance, what's the ROI on those massive data center investments? Stephan also notes working with AI is cognitively draining — the output flies in so fast your brain strains to keep up at a meta level. His take on Claude Code's moat: 50% the model, 40% the system prompts, 10% the CLI (plan mode, permissions) — switching to a competitor model gets you half the value at most.
