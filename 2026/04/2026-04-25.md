### 1. Model Wars: GPT-5.5 vs. Claude Code — Every (Context Window)

OpenAI's GPT-5.5 launch ignited an internal debate at Every between a Claude Code devotee and a Codex advocate, surfacing the clearest public articulation yet of where the two platforms diverge. The Claude camp argues that Anthropic's products — Claude Code CLI, Cowork, skills — are category-leading and that the company's recent consideration of removing Claude Code from the $20 Pro plan is "simple market economics" for a product in genuine demand. The Codex camp counters that Anthropic burns excessive compute and that OpenAI's infrastructure investment makes GPT-5.5 — designed as a token-efficient model — the smarter production choice. Both sides agree that Anthropic is compute-constrained while OpenAI over-invested and is now scrambling for users, and that the Claude Code desktop app is too buggy for daily use (only the CLI is production-grade). Elsewhere in the issue: the "not just X, it's Y" sentence construction has appeared 4× more frequently in large U.S. company documents since 2023, now an unmistakable AI tell. And a workflow for one-shotting product videos: screen-record the product, send to Opus for a storyboard, iterate, then have a coding agent build it in Remotion.

### 3. Freedom of Navigation: The Real Cost of the Hormuz Crisis — No Mercy No Malice

### 4. The 5 Silent Failures in Data Pipelines — SeattleDataGuy

A practical catalogue of the five ways data pipelines fail without throwing a single error — the failures that don't wake you up at 2 AM but persist until the CFO asks a question in a board meeting. (1) Schema drift: an upstream partner silently changes a headerless CSV and physician names land in the city field, with no error triggered. (2) Partial loads that look complete: APIs hit rate limits and stop returning data, yet report success — suspiciously round numbers like 10,000 rows or 2^16 are the tell. (3) Stale data: external partners' automation fails while your pipeline keeps succeeding, recycling last week's data for seven days before anyone notices. (4) Late-arriving dimensions: new categorical values stored in enumerators outside the database cause NULL propagation in reports and ML models. (5) Logic drift: hardcoded thresholds, date ranges, or tax rules that were correct when written silently misrepresent data as business context changes. The post is a sharp reminder that pipeline success status and data accuracy are not the same thing.
