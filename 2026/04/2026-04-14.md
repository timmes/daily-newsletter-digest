### 2. I Replaced Claude Code With a $45/Month Multi-Agent System
*Source: The AI Architect (Tyler Folkman) | Apr 12*

Claude Code's quality degraded sharply in February 2026: the Read:Edit ratio dropped from 6.6 to 2.0, and stop hook violations went from zero to 173 in 17 days. Boris Cherny (creator of Claude Code) confirmed the default effort level was quietly moved to 85% on March 3, dropping thinking depth 67%. Folkman built a replacement: a CRISPY multi-agent system via OpenCode (open-source) at ~$45/month vs $200/month for Claude Code Max. The system uses 5 specialized agents with intentionally different models: Architect (GLM 5.1, design only), Scout (Kimi K2.5, pure research), Coder (GLM 5.1, TDD mandatory), Reviewer (Qwen 3.6 Plus — *different model from coder to avoid shared blind spots*), and Verifier (GLM 4.7, runs actual tests). On a 1,364-line project, only one bug survived the pipeline. The key insight: model diversity in review outperforms using the best model everywhere, because the reviewer shares blind spots with the coder when they run on the same weights.

### 3. How LinkedIn Feed Uses LLMs to Serve 1.3 Billion Users
*Source: ByteByteGo | Apr 13*

LinkedIn replaced five separate content retrieval systems with a single LLM-powered dual encoder model delivering results in under 50ms at 1.3B user scale. The engineering details are worth studying: raw numbers in LLM prompts had near-zero correlation with relevance — converting popularity counts to percentile buckets (e.g., `<view_percentile>71</view_percentile>`) caused correlation to jump 30x and improved Recall@10 by 15%. Training exclusively on positively-engaged posts (filtering scrolled-past) reduced memory footprint 37%, improved throughput 40%, and made training 2.6x faster — more signal with less data. The Generative Recommender (GR) model treats your Feed history as a sequence, using causal transformer attention over 1,000+ interactions to understand temporal patterns. Late fusion architecture keeps count/affinity signals outside the quadratic self-attention computation, making the system viable at production scale. Takeaway: the bottleneck is never model architecture — it's feature encoding, data quality, and infrastructure.

### 4. Code Review Is the New Bottleneck for Engineering Teams
*Source: Engineering Leadership (Gregor Ojstersek) | Apr 13* ⭐ *Favorite*

With AI generating code faster than ever, code review has become the critical chokepoint. Data from LinearB's 2026 benchmarks (8.1M PRs, 4,800 teams across 42 countries): Agentic AI PRs have **5.3x longer pickup time** than unassisted PRs; AI-assisted PRs are **2.47x slower** to get reviewed. Root cause: "engineers dread reviewing AI-generated code and often procrastinate." Unlike building (which generates clear progress and dopamine), reviewing AI code requires full context reconstruction with no visible reward — and most companies don't track review speed as a KPI. Four high-impact solutions: write test cases before opening a PR; use AI to pre-review your own code; annotate confusing sections before assigning reviewers; add AI code review to CI/CD pipeline as an automatic check. Note: complex legacy codebases see smaller AI productivity gains — opinionated frameworks like Rails and Angular see the biggest boost because best practices are well-defined for the model.

### 5. The Folder Is the Agent
*Source: Every (Kieran Klaassen, Source Code) | Apr 13* ⭐ *Favorite*

Running 44 AI agents across multiple projects. Each one is just a model pointed at a folder. The key insight: the folder structure is the agent's memory, context, and interface — no complex orchestration frameworks needed. Each project folder has a CLAUDE.md with instructions, context files, and outputs. The model reads the folder, acts, and writes back. This approach enables 44 parallel agents with clean isolation, easy human inspection, and minimal infrastructure overhead. Complexity is the enemy: when something goes wrong, you can read the folder. When you want to change behavior, you edit a file. This pattern pairs with the trends toward CLAUDE.md as probabilistic context versus hooks as deterministic control — the folder is the "what," the hooks are the "how." The simplicity is the feature.

### 8. Meta Launches Muse Spark — Its First Hosted Model with 16 Tools

Benchmarks show it competitive with Opus 4.6 and GPT-5.4 on selected tests, but notably behind on Terminal-Bench 2.0 (agentic coding).
